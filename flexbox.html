Display: 💪

Display: flex asigna el display a la box model actual, por eso siempre se asigna al container de los elementos que queremos que se vuelvan flexibles.

  • Para que los elementos internos se muestren correctamente, mínimo debo definir un width o un max-width.
  • Para modificar el tamaño de los elementos internos (o qué tanto se estiran en la flexbox), hacerlo con padding o con width/height.
  • El display debe estar definido para poder usar todas las funciones de flex.
.container-flex
{
border:
1px solid red;
display:
flex;
}

.box
{
margin:
5px;
width:
25%;
background-color:
var(--second-color);
}

Flex-shrink Flex-grow

flex-shrink toma un número (por defecto 1) y encoge el elemento según el número (2 se reduce a la mitad, 4 a un cuarto, etc), comparando con los items a su alrededor.

Si hay espacio de sobra para que se extiendan todos los elementos en una misma línea, se va a ejecutar de forma incorrecta, nomás funcionando cuando se encoge y por un factor mucho más chico del que queríamos (por eso si defino width: 33%; no es tan evidente).

flex-grow en lugar de encogerlo, lo agranda, y para funcionar correctamente, el tamaño de los elementos no debe ocupar el 100% del width que pueden ocupar. Osea, todo al revés que el shrink (ej. no usar width: 33%;, sino width:25%).

  • OJO! La propiedad es para el item, no para el container que tiene el display: flex.
  • Si puedo evitar usar flex-grow con medidas absolutas mejor porque es un dolor de cabeza. Probar cambiando el width de box-grow-1.
  • También evitar poner muchos flex-grow o flex-shrink uno al lado del otro porque intentan relativizar sus tamaños entre sí y es un bardo.
  • Debería usarlos sólo si hay un item rebelde que quiero que se diferencie de los demás (y que sea responsive).
.box:nth-child(2)
{
/* Suponiendo un flexbox con 3 elementos, selecciono el segundo elemento */

flex-shrink:
2;
}

.item-number-2
{
/* Otra forma de hacerlo, creando una clase en el div del item a modificar */

flex-shrink:
2;

.container-A .box:nth-child(2)
{
/* OJO el width de .box!!! */
flex-grow:
4;
}

Flex-direction

Flex-direction no tiene mucha vuelta. Define la dirección de los elementos, que vayan en el eje X (por defecto: row) o en el eje Y.

También pueden revertirse la posición de todos los elementos en ambos ejes.

El cambiar de Eje hace que también las propiedades justify-content y align-items se reviertan, ojo con eso.

Align-content

Para que align-content funcione, necesito tener a los items distribuidos en varias lineas (usando flex-wrap).

Funciona de la misma forma que justify-content pero en vertical, distribuyendo a través de las diferentes "líneas" por las que se wrappean los elementos.

  • En caso de no tener un height definido para los boxes, estos se distribuyen por todo el espacio que pueden a causa de esta propiedad, la cual por defecto está en stretch.
  • Si uso flex-direction: column; esta propiedad pasa a comportarse como justify-content (y viceversa).

Align-content: stretch; (por defecto si los boxes no tienen height)

Order Align-self

order permite reordenar los elementos dentro de un container. Por defecto, todos los elementos tienen order: 0;. Reasignar el order a cualquiera de los elementos cambiará su ubicación en el display.

Reordenar con números positivos hará que el elemento se muestre más a la derecha, mientras que usar números negativos hará que el elemento se muestre más a la izquierda (porque se intentan reordenar con elementos sin un order definido, que por defecto toman el valor de 0).

align-self me permite hacer que un elemento específico ignore el align-items que le llega del container, alineandose a su manera.

  • Ambas propiedades aplican para el item, no para el container con el display: flex;.
  • Aunque no hace falta, align-self destaca más en casos en los que usamos también align-items.
.box:nth-child(2)
{
order:
-1;
}

.container
{
align-items
center;
}

.box-2
{
align-self:
stretch;
}
